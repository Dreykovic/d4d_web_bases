1. Balises et attributs

1.1. Les balises

  • Eléments principales des pages html
  • Celles-ci sont invisibles à l'écran pour vos visiteurs
  • Elles permettent à l'ordinateur de comprendre ce qu'il doit afficher
  • Les balises se repèrent facilement. Elles sont entourées de < chevrons >, c'est-à-dire des symboles < et >, comme ceci : <balise>
  • Elles indiquent la nature du texte qu'elles encadrent. Elles veulent dire par exemple : Ceci est le titre de la page , Ceci est une image Ceci est un paragraphe de texte etc.
  • On distingue deux types de balises :
    • les balises en paires <titre> Un texte</titre>
    • les balises orphelines.<image/> ou <image>

1.2. Les attributs

Les attributs sont les options des balises. Ils viennent les compléter pour donner des informations supplémentaires. L'attribut se place après le nom de la balise ouvrante et a le plus souvent une valeur, comme ceci :
<balise attribut="valeur">

2. Structure de base d'un fichier HTML

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>Titre</title>
  </head>
  <body></body>
</html>
Note

Vous noterez que les balises s'ouvrent et se ferment dans un ordre précis. Par exemple, la balise <html> est la première que l'on ouvre et c'est aussi la dernière que l'on ferme (tout à la fin du code, avec </html>). Les balises doivent être fermées dans le sens inverse de leur ouverture. Un exemple :

  • <html><body></body></html> : correct. Une balise qui est ouverte à l'intérieur d'une autre doit aussi être fermée à l'intérieur.
  • <html><body></html></body> : incorrect, les balises s'entremêlent.

2.1. Le doctype

<!DOCTYPE html>
La toute première ligne s'appelle le doctype. Elle est indispensable car c'est elle qui indique qu'il s'agit bien d'une page web HTML. Ce n'est pas vraiment une balise comme les autres (elle commence par un point d'exclamation). Vous
pouvez considérer que c'est un peu l'exception qui confrme la règle.

2.2. La balise <html>

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>Titre</title>
  </head>
  <body></body>
</html>

C'est la balise principale du code. Elle englobe tout le contenu de votre page.
Comme vous pouvez le voir, la balise fermante </html> se trouve tout à la fin du code !

2.3. L'en-tête <head> et le corps <body>

Une page web est constituée de deux parties :

  • L'en-tête <head> : cette section donne quelques informations générales sur la page comme
    • son titre,
    • l'encodage (pour la gestion des caractères spéciaux), etc.
    • Cette section est généralement assez courte.
    • Les informations que contient l'en-tête ne sont pas affichées sur la page,
  • Le corps <body> : c'est là que se trouve la partie principale de la page. Tout ce que nous écrirons ici sera affiché à l'écran. C'est à l'intérieur du corps que nous écrirons la majeure partie de notre code.

2.4. L'encodage (charset)

<meta charset="utf-8" />
Cette balise indique l'encodage utilisé dans votre fichier .html.
Le titre principal de la page: <title></title> Le titre ne s'affiche pas dans votre page mais en haut de celle-ci (souvent dans l'onglet du navigateur).

2.5. Les commentaires

<!-- Ceci est un commentaire -->
Vous pouvez le mettre où vous voulez au sein de votre code source : il n'a aucun
impact sur votre page, mais vous pouvez vous en servir pour vous aider à vous
repérer dans votre code source (surtout s'il est long).

3. Organiser son texte

Nous allons voir successivement dans cette partie :

  • comment rédiger des paragraphes ;
  • comment structurer sa page avec les titres ;
  • comment donner de l'importance à certains mots de son texte ;
  • comment organiser les informations sous forme de listes.

3.1. Paragraphes

Le langage HTML propose justement la balise <p> pour délimiter les paragraphes.

<p>Bonjour et bienvenue sur mon site !</p>

3.2. Sauter une ligne

il existe une balise "Aller à la ligne" ! C'est une balise orpheline qui sert juste à indiquer qu'on doit aller à la ligne : <br/>.

<p>
  123 Rue de l'Exemple<br>
  Ville Exemple<br>
  Pays Exemple
</p>

3.3. Les titres

En HTML, on est verni, on a le droit d'utiliser six niveaux de titres différents. On a donc six balises de titres différentes :

  • <h1> </h1> : signifie « titre très important ». En général, on s'en sert pour afficher le titre de la page au début de celle-ci.
  • <h2> </h2> : signifie « titre important ».
  • <h3> </h3> : pareil, c'est un titre un peu moins important (on peut dire un « sous-titre » si vous voulez).
  • <h4> </h4> : titre encore moins important.
  • <h5> </h5> : titre pas important.
  • <h6> </h6> : titre vraiment, mais alors là vraiment pas important du tout.

3.4. La mise en valeur

  • <em> </em> Mettre un peu en valeur
  • <strong></strong>: Mettre Beaucoup en valeur.
    Il y'en a plein d'autre. Mais habituellement le CSS est plus utilisé pour la mise en valeur.

3.5. Les listes

Les listes nous permettent souvent de mieux structurer notre texte et d'ordonner nos informations.
Nous allons découvrir ici deux types de listes :

  • les listes non ordonnées ou listes à puces ;
<ul>
  <li>Fraises</li>
  <li>Framboises</li>
  <li>Cerises</li>
</ul>
  • les listes ordonnées ou listes numérotées ou encore énumérations.
<ol>
  <li>Fraises</li>
  <li>Framboises</li>
  <li>Cerises</li>
</ol>

3.6. Créer des liens

Pour faire un lien, la balise que nous allons utiliser est très simple à retenir : <a>. Il faut cependant lui ajouter un attribut, href, pour indiquer vers quelle page le lien doit conduire.

<a href="https://github.com">Github</a>
<a href="page.html" title="Vers la page">Github</a>

4. Autres composants HTML

4.1. Les images

Pour insérer une image dans une page HTML, nous utilisons la balise <img/>
C'est une balise de type orpheline (comme <br />).
La balise doit être accompagnée de deux attributs obligatoires :

  • src : il permet d'indiquer où se trouve l'image que l'on veut insérer. Vous pouvez soit mettre un chemin
    absolu (ex. : http://www.site.com/fleur.png), soit mettre le chemin en relatif (ce qu'on fait
    le plus souvent). Ainsi, si votre image est dans un sous-dossier images, vous devrez taper :
    src="images/fleur.png".
  • alt : cela signifie « texte alternatif ». On doit toujours indiquer un texte alternatif à l'image, c'est-à-dire un court texte qui décrit ce que contient l'image. Ce texte sera affiché à la place de l'image si celle-ci ne peut pas être téléchargée (cela arrive), ou dans les navigateurs de personnes handicapées (non-voyants) qui ne peuvent malheureusement pas « voir » l'image. Cela aide aussi les robots des moteurs de recherche pour les recherches d'images. Pour la fleur, on mettrait par exemple : alt="Une fleur".

Voici un exemple d'insertion d'image :

<img src="images/montagne.jpg" alt="Photo de montagne" />

4.2. Tables en HTML

4.2.1. Généralités

Les tables en HTML sont utilisées pour afficher des données tabulaires, telles que des listes de données, des calendriers, etc. Une table est composée de lignes (éléments <tr>) et de cellules (éléments <td> pour les cellules de données et <th> pour les cellules d'en-tête).

<table>
  <thead>
    <tr>
      <th>Entête 1</th>
      <th>Entête 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Donnée 1</td>
      <td>Donnée 2</td>
    </tr>
    <!-- Ajoutez d'autres lignes et cellules ici -->
  </tbody>
</table>

4.2.2. Attributs des éléments de table

  1. colspan : Permet de spécifier le nombre de colonnes qu'une cellule devrait occuper.

Exemple : 

<td colspan="2">Cellule occupant deux colonnes</td>
  1. rowspan : Permet de spécifier le nombre de lignes qu'une cellule devrait occuper.

Exemple :

<td rowspan="2">Cellule occupant deux lignes</td>
  1. border : Définit l'épaisseur de la bordure autour de la table.

Exemple :

<table border="1">

4.3. Formulaires en HTML

4.3.1. Généralités

Les formulaires en HTML sont utilisés pour collecter des informations auprès des utilisateurs. Les formulaires sont constitués d'éléments tels que les zones de texte, les boutons radio, les cases à cocher, les menus déroulants, etc., qui permettent aux utilisateurs d'entrer et de soumettre des données.
Voici un exemple de structure de base d'un formulaire en HTML :

<form action="/action_page.php" method="post">
  <label for="fname">Prénom :</label>
  <input type="text" id="fname" name="firstname"><br><br>
  <label for="lname">Nom :</label>
  <input type="text" id="lname" name="lastname"><br><br>
  <input type="submit" value="Envoyer">
</form>

4.3.2. Attributs des éléments de formulaire

  1. action : Spécifie l'URL vers laquelle envoyer les données du formulaire.

Exemple :

<form action="/mon-script.php" method="post">
  1. method : Définit la méthode HTTP à utiliser pour envoyer les données du formulaire. Les valeurs couramment utilisées sont "get" et "post".

Exemple :

<form action="/action_page.php" method="post">
  1. name : Donne un nom à l'élément de formulaire, utilisé pour référencer les données côté serveur.

Exemple :

<input type="text" name="firstname">
  1. type : Spécifie le type de champ de formulaire. Les valeurs courantes incluent "text", "password", "checkbox", "radio", "submit", "reset", etc.

Exemple :

<input type="text">
  1. value : Définit la valeur initiale d'un champ de formulaire.

Exemple :

<input type="text" value="Valeur initiale">
