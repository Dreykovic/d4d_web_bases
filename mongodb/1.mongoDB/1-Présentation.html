Introduction

Une base de données est un environnement pour stocker les données d’une manière sécurisée, fiable et facilement disponible. Il existe deux(2) types de base de données en général:

  • Les bases de données relationnelles: Elles sont utilisées pour stocker des données structurées. Ces dernières sont organisées en lignes et colonnes. Comme example, on peut citer: postgreSQL, mySQL, Oracle database, ...
  • Les bases de données non-relationnelles ou NoSQL: elles ont la particularité de stocker des données non structurées. Ces données sont souvent des données complexes et diverses comme: les catalogues de produits, les journaux(logs), les interactions avec l’utilisateur, la statistique et autres. Les exemples de base de données non relationnelles sont: Amazon dynamoDB, cassandra, mongoDB, etc.

MongoDB est l’une des plus populaires des solutions NoSQL. Elle peut être utilisée pour stocker des données non structurées, semi structurées ou structurées. MongoDB est utilisée par des compagnies comme: Google, Facebook, Adobe, Cisco, eBay et autres. Elle dispose d’une version gratuite(community edition) et d’une version payante(entreprise edition).

Avantages de MongoDB

  • Schéma flexible : Permet d'ajouter de nouveaux champs sans altérer le schéma existant.

  • Évolutivité : Conçu pour le déploiement sur des clusters de serveurs, permettant une mise à l'échelle horizontale facile.

  • Performance : Les opérations de lecture et d'écriture sont rapides grâce à la capacité de stockage des données sous forme de documents.

  • Requêtes riches : Offre une variété de méthodes de requête, y compris des requêtes imbriquées, des expressions régulières, etc.

  • Intégration facile avec des langages de programmation : La représentation des données sous forme de JSON facilite l'intégration avec de nombreux langages de programmation.

Limites de MongoDB

  • Pas adapté aux transactions ACID complexes : Bien que MongoDB prenne en charge les transactions sur un seul document, il n'est pas aussi adapté que les SGBDR pour les transactions ACID (Atomicité, Cohérence, Isolation, Durabilité) complexes impliquant plusieurs documents ou collections.

  • Consommation de mémoire : Peut consommer plus de mémoire que les SGBDR en raison de sa conception sans schéma fixe et de son utilisation de l'indexation en mémoire.

  • Pas adapté aux applications nécessitant des jointures complexes : Les jointures entre collections sont moins efficaces dans MongoDB par rapport aux SGBDR, ce qui peut poser des problèmes pour les applications nécessitant des requêtes complexes impliquant de multiples collections.

Différences avec les SGBD SQL

MongoDB est une base de données NoSQL, ce qui signifie qu'elle diffère fondamentalement des systèmes de gestion de bases de données relationnelles (SGBDR) tels que MySQL, PostgreSQL, etc. Voici quelques différences clés :

  • Modèle de données : Les bases de données relationnelles utilisent un modèle tabulaire avec des tables, des lignes et des colonnes, tandis que MongoDB utilise un modèle de données orienté document, stockant les données dans des documents JSON (JavaScript Object Notation).

  • Langage de requête : Les SGBDR utilisent SQL (Structured Query Language) pour interagir avec la base de données, tandis que MongoDB utilise un langage de requête basé sur des objets JSON appelé Query Language.

  • Schéma flexible : Les bases de données relationnelles ont un schéma fixe définissant la structure des données, tandis que MongoDB permet un schéma flexible, ce qui signifie que les documents dans une collection peuvent avoir des structures différentes.

Domaines d’usage de MongoDB

  1. Big data
  2. Gestion et diffusion de contenu
  3. Infrastructure mobile et sociale
  4. Gestion des données utilisateur
  5. Centre de données
